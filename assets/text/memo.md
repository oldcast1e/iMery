## 1. 태그 기능 추가
1.  Quick Tag는 DB에 Tag 칼럼으로 생성/저장되며, Tag_High, Tag_Middle, Tag_Low로 구분됩니다.
2. 사용자가 [작품], [공간], [경험]의 대분류 대분류 태그를 클릭하여 중분류 태그, 소분류 태그를 선택할 수 있도록 구성합니다.
3. [대분류] 태그는 필수로 선택할 필요가 없으며, 선택하지 않을 경우, 태그 정보를 추가하지 않습니다.
4. DB에는 최대 5개의 태그를 저장할 수 있도록 하며, 사용자가 5개 초과로 선택 시, 알람을 띄워 사용자에게 5개 이하의 태그를 선택하도록 합니다. 이미 5개의 태그를 선택한 경우, 추가로 태그를 선택할 수 없습니다.
5. 사용자는 [작품], [공간], [경험]의 대분류를 태그를 선택할 시, 중분류 태그까지는 선택해야합니다. 
    - 사용자는 [중분류] 태그까지만 선택해도 좋으며, 태그한 값까지 데이터 베이스에 저장됩니다.
    - 사용자는 [중분류] 태그까지만 선택 시, [소분류] 태그 정보는 저장하지 않습니다.
    - 사용자가 [중분류] 태그까지만 선택한 경우, [중분류] 태그를 DB에 저장합니다.
6. 사용자 선택한 태그의 가장 하위 태그만 저장합니다.
    - 예시 : [작품] -> [압도적]  =>  [압도적] 태그 저장
    - 예시 : [공간] -> [기쁨] -> [황홀경] => [황홀경] 태그 저장
    - 예시 : [경험] -> [평화] => [평화]  태그 저장
7. 태그 정보는 반응형으로 제작합니다. 모든 태그가 처음부터 사용자에게 보이지 않습니다.
    - 화면 비율을 고려하여 태그를 출력합니다.
    - 예시: 작품 등록 시, [작품] (대분류)태그 버튼 클릭 -> [압도적]~[분위기]까지의 중분류 태그 출력
    - 예시 : [압도적] 중분류 태그 -> [웅장함]~[전율적]까지의 소분류 태그 출력
    - 선택한 태그(버튼)는 우측의 x 버튼을 통해 삭제할 수 있도록 합니다. 
    - 단, 상위 태그를 삭제시, 하위태그도 화면에서 삭제되어야 합니다.
    - 현재 앱 디자인과 어울리도록 디자인하세요.
8.  태그 정보는 작품 등록 시, DB에 저장됩니다.
9. 키워드의 구성은 아래와 같습니다. 단, 설명은 제외하고 키워드만 사용하세요.
    - 예시: [작품] -> [압도적] -> [웅장함]
    - 예시: [공간] -> [기쁨] -> [황홀경]
    - 예시: [경험] -> [평화] -> [고요]
10. 선택한 최종 태그(6번 참조) [저장된 작품] 페이지에서 확인할 수 있도록 합니다.
    -  [저장된 작품] 페이지의 작품 등록일/작가 텍스트 아래에 배치합니다.
    - 태그 정보는 왼쪽 정렬하며, 각 태그의 box는 동일합니다. 메인 디자인 형식을 따르세요.

## 1. UI 개선
- [작품 폴더] 페이지에서 class="absolute inset-0 bg-white/40 backdrop-blur-sm"와 같은 버튼을 클릭 시(예: 전체 작품, 북마크 등) 뜨는 페이지의 UI 개선 필요
- 작품의 태그가 너무 크게 반영되어, 전체적인 UI 디자인 구조를 망가트림. (첨부 이미지 참조)
- 작품 태그를 [작품 폴더] 하위 페이지의 레이아웃에 맞춰서 동적으로 반영되도록 수정하며, [작가 이름], [업로드 날짜] 하단에 태그를 추가, 디자인 분위기를 맞춰야함.
- 단, [업로드 날짜] 텍스트보다 태그 box가 작게 디자인할 것. 변경된 UI 디자인은 [작품 폴더] 모든 하위 페이지에 적용되어야함.
- [작품 폴더] 하위 페이지(예: 전체 작품, 북마크 등) 삭제/수정 버튼의 디자인이 전체적인 앱 디자인과 어울리지 않음. 버튼 UI 개선이 필요함.

- [← 폴더로 돌아가기] 버튼이 앱의 전반적인 디자인 분위기와 어울리지 않으며, [작품 폴더] 하위 페이지(예: 전체 작품, 북마크 등)의 제목(예:전체 작품) 텍스트와 불필요하게 여백이 존재함. 버튼 UI를 개선하고, 불필요한 여백 공간을 제거하여 적절한 여백을 반영해.

- "별이 빛나는 밤 / 빈센트 반 고흐 • 2026.01.21"와 같은 [작품 폴더] 하위 페이지(예: 전체 작품, 북마크 등)의 작품 설명에 배경이 없어 글씨가 잘 보이지 않아. 두 텍스트를 포함하는 불투명한 배경을 추가하여, 텍스트가 잘 보이도록 강조해. 단, 이 배경색의 부여는 다른 작품 설명 버튼의 레이아웃에 통일성 있게 동일하게 적용해야해.

- 수정/삭제 버튼이 불필요하게 크고, 색상이 전반적인 앱 디자인 컬러와 달라. 두 버튼의 크기를 축소하고, 배경색을 살짝 흐리게 변경해. 삭제 버튼의 색상은 메인 색상(R: 35 / G:84 / B:157)으로 변경하자.

## 3. 음악 기능 추가
- 사용자가 [홈]-[저장된 작품] 페이지에 들어갈 경우, DB의 posts/music_url에 접근하여 음악을 저장/재생하는 기능을 추가. (저장 예시 : https://imery.s3.ap-southeast-2.amazonaws.com/memphis-trap-wav-349366.wav)
- 음악은  [홈]-[저장된 작품] 페이지에 들어갈 경우, 자동으로 재생되어야하며 약 10초의 노래가 반복됨
- 음악 아이콘은 [홈]-[저장된 작품]에서 공유 아이콘 왼쪽에 위치하며, 토글 기능(클릭 시, 재생 중단. 다시 클릭 시 재생 시작)을 가지도록 수정.
- 음악 아이콘은 AI 기능임을 시각적으로 알 수 있도록, [AI 분석 버튼]과 비슷한 레이아웃 디자인을 가지록 디자인 할 것.
- 음악이 재생되는 경우, 아이콘이 재생 중임을 알 수 있도록 아이콘의 색상이 변하게 디자인.
- posts/music_url에 값이 존재하지 않을 때 클릭 시, "AI가 노래를 생성 중입니다!" 알림 창을 띄울 것. 단, 화면 창의 레이아웃에 맞춰 반응형 디자인을 적용할 것.이후 알림창이 자연스럽게 페이딩되도록 할 것.
