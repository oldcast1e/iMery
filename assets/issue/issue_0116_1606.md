# v0.1.0 업데이트 가능성 분석 보고서

**분석 일시**: 2026.01.16 16:06
**분석 대상**: `assets/text/update.md` (기능 명세) vs 현재 코드베이스 (`server/index.js`, `App.jsx`)

## 1. 종합 요약
**결론**: **실현 가능함 (Feasible)**. 단, Frontend 개선에 앞서 **Backend(DB 및 API)의 대대적인 업데이트가 선행되어야 함**.
현재 백엔드는 기본적인 CRUD와 친구 요청 기능만 구현되어 있어, 기획된 [아카이브] 및 [마이페이지-활동] 기능을 지원하기 부족함.

---

## 2. 상세 분석 (Gap Analysis)

### A. 검색창 제거 및 작품 추가 버튼 (Frontend Only)
- **상태**: ✅ **실현 가능**
- **분석**: 기존 `BottomNav`의 중앙 버튼 UI 변경 및 라우팅 로직 수정으로 즉시 구현 가능.

### B. [아카이브] (구 커뮤니티) 개편
#### 1. 피드 탭 (북마크 기능)
- **상태**: ⚠️ **Backend 수정 필요**
- **현재**: 북마크가 `localStorage`에만 저장됨 (기기 간 동기화 불가).
- **필요 작업**:
    - **DB**: `Bookmarks` 테이블 생성 (`user_id`, `post_id`, `created_at`).
    - **API**: `POST /bookmarks`, `DELETE /bookmarks`, `GET /users/:id/bookmarks` 구현 필요.

#### 2. 달력 탭 (캘린더 뷰)
- **상태**: ✅ **실현 가능**
- **분석**: `GET /posts` API가 `created_at`을 반환하므로, 프론트엔드에서 날짜별 그룹핑(Grouping) 로직만 구현하면 됨. API 수정 불필요.

#### 3. 친구 탭 (관리 기능)
- **상태**: ⚠️ **API 추가 필요**
- **현재**: 친구 목록 조회(`GET`)는 가능.
- **필요 작업**:
    - **API**: 친구 삭제(팔로우 취소)를 위한 `DELETE /friends/:id` API 부재. 추가 필요.

### C. 검색 기능 개편 (Frontend Only)
- **상태**: ✅ **실현 가능**
- **분석**: 현재 Client-side Filtering 구조이므로, 프론트엔드 로직 수정만으로 홈 화면 내 실시간 검색 구현 가능.

### D. [마이] 페이지 개편
#### 1. 프로필 (사진, 소개, 통계)
- **상태**: ❌ **Backend 대공사 필요**
- **현재**: `Users` 테이블에 `nickname`만 존재. 프로필 사진, 한줄 소개 컬럼 없음.
- **필요 작업**:
    - **DB**: `Users` 테이블에 `profile_image_url`, `bio` 컬럼 추가.
    - **API**: 프로필 수정을 위한 `PUT /users/profile` (이미지 업로드 포함) 구현 필요.
    - **API**: 팔로워/팔로잉 수 카운트 API 필요 (현재는 친구 목록을 다 불러와서 세야 함 -> 비효율적).

#### 2. 나의 활동 (북마크/좋아요/댓글 모아보기)
- **상태**: ⚠️ **API 추가 필요**
- **분석**:
    - **좋아요**: `GET /users/:id/likes` 존재 (Post ID만 반환). 화면 표시를 위해 Post 상세 정보까지 Join해서 반환하는 쿼리 개선 필요.
    - **댓글**: "내가 쓴 댓글"을 모아보는 API 없음. `SELECT * FROM Comments WHERE user_id = ?` API 추가 필요.

#### 3. 계정 설정
- **상태**: ❌ **기능 부재**
- **필요 작업**: 비밀번호 변경(`PUT`), 회원 탈퇴(`DELETE`) API 신규 개발 필요.
